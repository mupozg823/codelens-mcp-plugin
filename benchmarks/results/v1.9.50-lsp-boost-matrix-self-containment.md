# P1-4 end_line-containment lsp_boost matrix (self, v1.9.50, 2026-04-20)

**Binary:** `target/release/codelens-mcp` with migration-7 DB schema +
`SymbolInfo.end_line` threaded through parse, persistence, and rank.
**Repo:** `codelens-mcp-plugin` (self)
**Refs backend:** LSP (cold) + tree-sitter union
**Dataset:** `benchmarks/lsp-boost-dataset-self.json` (10 caller queries)
**Script:** `benchmarks/lsp-boost-spotcheck.py --dataset ...`

## Headline

| Arm                      | weight | hits@K | MRR    | ΔMRR vs baseline |
| ------------------------ | ------ | ------ | ------ | ---------------- |
| baseline                 | –      | 1/10   | 0.0500 |                  |
| file-level               | 0.25   | 3/10   | 0.0670 | +34%             |
| per-symbol (proximity)   | 0.25   | 6/10   | 0.0934 | +87%             |
| per-symbol (containment) | 0.25   | 8/10   | 0.1145 | +129%            |

Per-symbol _containment_ improves on the proximity-only variant by
surfacing two queries that previously missed entirely:

| query                        | per-sym proximity | containment |
| ---------------------------- | ----------------- | ----------- |
| LspRequest                   | —                 | 9           |
| default_lsp_command_for_path | —                 | 21          |
| prune_to_budget              | 13                | 5 (▲8)      |

Two minor regressions (rank_symbols 5→7, RankingContext 19→23) are
the expected noise of redistributing full-strength boosts from
"close-to-declaration" symbols to actual span-containing callers.

## What changed

1. `SymbolInfo.end_line` is now a first-class field on both
   `SymbolInfo` and `ParsedSymbol`, populated at parse time from
   tree-sitter's `Node::end_position().row + 1`.
2. A new DB migration (`7`) adds the `end_line INTEGER NOT NULL
DEFAULT 0` column to `symbols`. Pre-migration rows read back as 0
   and fall back to `line` so the P1-4 blend keeps working on stale
   caches.
3. `SymbolRow`, `NewSymbol` carry `end_line`. All `INSERT` and
   `SELECT` sites that touch the symbol column list were extended.
4. `lsp_proximity_factor` now takes `(ref_lines, symbol_line,
symbol_end_line)` and returns `1.0` for any ref inside
   `[symbol.line, symbol.end_line]`; outside the span it decays
   linearly over `LSP_PROXIMITY_WINDOW_LINES = 250`. The gate rescue
   and the blend reuse the same factor.
5. A new engine test
   (`lsp_signal_containment_beats_nearby_non_container`) pins the
   containment contract. Engine 289 → 290, MCP `--features http`
   499/499.

## Honest limits

- 2 queries still miss (`LspSessionPool`, `SymbolInfo`). Their
  expected callers are in very large files where other symbols
  _also_ contain refs for the same query target — the blend cannot
  tell "best container" from "a container" without a per-ref
  weighting signal.
- LSP is still cold on CLI one-shot (0 refs). All signal here is
  tree-sitter-unioned. An HTTP daemon with prewarm would change the
  ref distribution — the wiring is ready for that as-is.
- Dataset is 10 queries, 1 repo. zod/flask datasets + multi-repo
  matrix are the next falsification steps.

## Where this leaves P1-4

- Default weight stays at 0.25; containment lifts precision without
  needing a heavier cap.
- Legacy DB caches keep serving correct answers (fallback = line)
  until they are refreshed.
- The blend signals now separate "lives in a boosted file" from "is
  the container of a ref" — the cleanest architectural cut the
  caller-focused dataset asked for.
