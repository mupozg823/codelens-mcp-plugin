# P1-4 auto-attach lsp_boost matrix (v1.9.50, 2026-04-20)

Follow-up to the `--warm-lsp` matrix
(`v1.9.50-lsp-boost-http-matrix.md`). The HTTP bench runner now has
an `--auto-attach` mode that mimics what a normal agent session does:
set `CODELENS_LSP_AUTO=true`, call `prepare_harness_session` once,
wait for the fire-and-forget LSP prewarm to settle, then run the
measurement arms — _no_ explicit per-query
`find_referencing_symbols(use_lsp=true)` calls.

If auto-attach gets close to the explicit prewarm numbers, agent
sessions that already call `prepare_harness_session` at start-up get
the per-language uplift for free.

## Headline

| Repo  | Baseline         | Per-ref CLI      | HTTP warm-LSP    | HTTP auto-attach     | ΔMRR vs warm |
| ----- | ---------------- | ---------------- | ---------------- | -------------------- | ------------ |
| self  | 1/10, MRR 0.0500 | 7/10, MRR 0.1264 | 7/10, MRR 0.1294 | 7/10, MRR 0.1221     | −6%          |
| flask | 2/10, MRR 0.0500 | 3/10, MRR 0.0553 | 4/10, MRR 0.0893 | **4/10, MRR 0.1053** | **+18%**     |
| zod   | 4/10, MRR 0.0932 | 4/10, MRR 0.0713 | 4/10, MRR 0.0713 | 4/10, MRR 0.0713     | flat         |

- **self** sheds 0.007 MRR versus warm-lsp — noise, well inside
  per-run variance. rust-analyzer on a freshly refreshed self index
  was already ~saturated by tree-sitter.
- **flask** beats explicit warm. Auto-attach gives pyright more
  contiguous indexing time (45 s of fire-and-forget before any
  query lands) than per-query warm-lsp, which races the indexer.
  Two expected callers (`Blueprint → Blueprint` at rank 3,
  `Scaffold → App` at rank 2) now land in top-K; `send_file →
send_from_directory` stays at rank 7; `render_template →
stream_template` is surfaced at rank 13.
- **zod** is unchanged — the dataset asks for thin one-ref
  wrappers, which per-ref weighting correctly refuses to promote
  above multi-ref containers regardless of ref precision.

## What it costs

- `--auto-attach-wait 45` (flask / zod default) gates the arms
  until the fire-and-forget prewarm has had time to index. In a
  real agent flow this overlap is the agent's own reasoning time,
  so wall-clock is free — but the bench has to wait.
- On self we use `--auto-attach-wait 30` because rust-analyzer
  indexes Rust crates faster than pyright does Python.

## Why this matters

An agent that calls `prepare_harness_session` at the start of a
session and then queries `get_ranked_context(lsp_boost=true)` for a
real task gets the flask-style uplift without any new tool calls.
The only requirement is that `CODELENS_LSP_AUTO` is not disabled.

## Honest limits

- Datasets are still 10 queries each. Direction-only.
- Flask's `prepare_harness_session` response carried
  `lsp_auto_attach: null` in this run — the wall-clock uplift is
  real, but the telemetry needs a small fix so callers can see
  which languages actually fired. Follow-up.
- Zod lane remains dataset-bound.
