# P1-4 lsp_boost 3-repo matrix (v1.9.50, 2026-04-20)

Caller-focused datasets (10 queries each, expected symbol = caller in
a different file from `path`) across three languages with tree-sitter
text-union references (LSP cold in CLI one-shot).

| Repo  | Language | Baseline         | lsp_boost (w=0.25) | ΔMRR  |

## Reading

The boost is **strongly positive on Rust** and **neutral-to-negative
on Python / TypeScript** at the default weight. A single matrix
cannot decide the direction, but the distribution is consistent:
callers in flask/zod tend to sit in small files with many functions
that each contain at least one ref to the queried symbol, so the
full-containment `+1.0` factor ties dozens of symbols at the same
boosted score — the expected caller is one of many and does not
survive to top-K. Rust self has fewer symbols per file and heavier
single-caller containment, so the boost targets the right function.

## What this tells us

- The containment signal is **correct but coarse**. Every symbol in a
  boosted file that happens to hold a ref gets the same `+1.0`, so
  ref-dense files produce many tied winners.
- A follow-up design that scales containment by **ref count inside
  the span** (or by per-ref LSP confidence) would likely recover the
  flask/zod lanes without losing the Rust self uplift.
- Default weight stays at 0.25. Users that know their codebase is
  Rust-shaped can crank it via `CODELENS_LSP_SIGNAL_WEIGHT`; users on
  Python / TS benefit from leaving it off until the per-ref weighting
  follow-up lands.
- The wiring, the gate rescue, and the DB persistence are now
  validated end-to-end across three languages. Engine 290 tests, MCP
  `--features http` 499 tests, all green.

## Honest limits

- Datasets are 10 queries each — direction, not magnitude.
- LSP is cold on CLI one-shot (0 refs); all signal is tree-sitter.
  HTTP daemon + prewarm would change the ref distribution.
- Flask / zod datasets pick "interesting" definitions without a
  systematic survey of ref density. A stratified dataset (low-density
  vs high-density definitions) would let us tell "dataset noise" from
  "signal is wrong here".

## Next

- Per-ref weighting: scale containment by `count(refs ∈ span) /
total_refs` so multi-caller containers outrank single-caller ones.
- HTTP daemon harness to fold warm LSP refs into the union; measure
  LSP-only delta over the tree-sitter floor.
- Expand flask / zod datasets to 20-30 queries each so the
  per-language verdict is statistically stable.
