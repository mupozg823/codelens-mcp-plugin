uniform cap pulls competing symbols up with the target caller.

## What changed vs the file-level arm

The per-symbol refactor replaces
`RankingContext.lsp_boost_files: HashSet<String>` with
`lsp_boost_refs: HashMap<String, Vec<usize>>` — a map from each ref'd
file to the **sorted line numbers** that the reference probe
(LSP ∪ tree-sitter) returned. Inside `rank_symbols`:

1. `lsp_proximity_factor(ref_lines, symbol.line)` finds the smallest
   `ref_line ≥ symbol.line` and converts the line distance into a
   `0.0..=1.0` factor via a linear decay over
   `LSP_PROXIMITY_WINDOW_LINES`. A ref exactly on the symbol's
   declaration line yields 1.0; a ref 250+ lines below yields 0.0.
2. The blend now contributes `100 * weights.lsp_signal * factor`
   instead of the old uniform `100 * weights.lsp_signal`.
3. The gate rescue also keys on the proximity factor, so a zero-text
   symbol is only rescued when a ref plausibly lives inside its body.

The window is 250 lines rather than 100 because the top 5% of
functions in this repo run to 150-220 lines; a 100-line window
silently dropped the caller for `rank_symbols` on this dataset. A
future revision can replace the heuristic with the symbol's true
`end_line` threaded through `SymbolInfo` at tree-sitter extraction
time.

Tests added:

## Honest limits

- Four queries still miss. Inspection of `LspRequest` and
  `default_lsp_command_for_path` shows the expected caller symbol is
  in a big file but the ref lines cluster near line ~300 while the
  caller `find_referencing_symbols` _starts_ at line 190 — within the
  window — and _ends_ at ~437. The proximity factor works correctly;
  the caller just doesn't win against other symbols in that pool
  whose line sits closer to a ref. A proper `end_line` plumb-through
  would let us score on _containment_, not just proximity.
- Elapsed time is ~1.3 s per query — the tree-sitter text search
  over 128 refs dominates. HTTP daemon + prewarm would cut this.
- LSP (rust-analyzer) is still cold on CLI one-shot and returns 0
  refs in ~7 ms; the signal here is almost entirely tree-sitter.
