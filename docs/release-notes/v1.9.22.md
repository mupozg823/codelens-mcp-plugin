# v1.9.22 Release Notes

## Highlights

- **SCIP precise backend**: full `PreciseBackend` trait implementation using Sourcegraph SCIP index files
- **SCIP-aware `find_referencing_symbols`**: auto-detects SCIP index and provides type-aware cross-file references
- **Training data pipeline expansion**: auto-clone registry with 12 repos across 7 languages
- **693 tests passing** across all feature configurations

## New Features

### SCIP Backend (`--features scip-backend`)

Feature-gated implementation of the `PreciseBackend` trait backed by SCIP (Source Code Intelligence Protocol) index files:

- **find_definitions**: type-aware definition lookup with 3-tier symbol resolution (exact location → same-file → global)
- **find_references**: cross-file reference tracking with definition/reference role classification
- **hover**: documentation from SCIP symbol info + override docs
- **diagnostics**: SCIP diagnostic extraction with severity mapping
- **Auto-detection**: checks `index.scip`, `.scip/index.scip`, `.codelens/index.scip`
- 7 unit tests covering all trait methods

### SCIP Integration in `find_referencing_symbols`

When `scip-backend` feature is enabled and a SCIP index exists:

- Automatically tried before tree-sitter fallback (after oxc_semantic for JS/TS)
- Returns `backend: "scip"` with confidence 0.98
- Falls through to tree-sitter if SCIP has no index for the file

### Training Data Pipeline

- `--auto-clone` flag with 12-repo registry (Flask, Django, curl, gin, echo, gson, guava, ripgrep, axum, Express, Jest, requests)
- Automatic shallow clone + caching in `--clone-dir`
- Languages: Python, C, Go, Java, Rust, JavaScript, TypeScript
