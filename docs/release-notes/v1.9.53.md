# v1.9.53

## Summary

v1.9.53은 v1.9.52의 `manual harness`를 `release-ready automated harness`로 끌어올린 릴리스다.
기본 동작은 그대로 `advisory`로 유지하고, 필요한 환경에서만 `strict` coordination mode를 켤 수 있게 했다.

핵심은 세 가지다.

- `benchmarks/harness/release-harness-runner.py` 추가
- trusted HTTP `refactor-full` mutation에만 적용되는 opt-in strict coordination
- `<usage>` 블록 + session metrics를 교차검증하는 usage-drift artifact 표준화

## Automated Harness

새 release harness runner는 아래 single-flow를 한 번에 실행하거나 checkpoint 재사용으로 이어받는다.

1. `preflight`
2. `worker_scan`
3. `orchestrator`
4. `evaluator`
5. `orchestrator_repair` / `evaluator_repair` 최대 1회
6. `independent_signoff`

각 child stage의 `run-manifest.json`과 artifact 경로는 부모 run manifest에 연결된다.
릴리스 통과 artifact는 다음 네 파일로 고정했다.

- `usage-drift.json`
- `usage-drift.md`
- `independent-signoff.json`
- `independent-signoff.md`

Independent evaluator verdict가 없거나 self-grade와 불일치하면 전체 run은 `fail`로 종료된다.

## Strict Coordination Mode

새 runtime option:

```bash
--coordination-mode advisory|strict
```

동일 설정 env:

```bash
CODELENS_COORDINATION_MODE=advisory|strict
```

기본값은 `advisory`다. `strict`는 아래 범위에만 적용된다.

- trusted non-local HTTP session
- `refactor-full` mutation surface

Strict mode에서는 다음이 runtime hard block이 된다.

- mutation target path가 최신 preflight `target_paths`에 없음
- mutation target path가 현재 session의 active `claim_files`에 없음
- 최신 preflight가 `overlapping_claims`를 보고함

Structured failure kind:

- `MissingClaim`
- `ClaimPathMismatch`
- `OverlappingClaimsBlocked`

## Verification

릴리스 게이트에 아래 검증을 추가했다.

```bash
cargo test -p codelens-mcp --features http
python3 -m unittest discover -s benchmarks/harness/tests -p 'test_*.py'
python3 scripts/agent-contract-check.py --strict
```

의미는 다음과 같다.

- Rust HTTP tests: strict coordination mutation block 회귀 방지
- Python harness tests: release runner stage ordering, checkpoint reuse, usage parser, independent disagreement 회귀 방지
- agent contract check: repo-shipped agent prompt와 실제 tool surface 계약 검증
