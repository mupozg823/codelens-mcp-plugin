# v1.9.57

## Summary

`v1.9.57`은 control-plane hardening release다. 목표는 public MCP 이름과
payload shape는 그대로 유지하면서, suggestion / prompt / bootstrap routing이
같은 canonical registry를 보게 만드는 것이다.

이번 라인에서 바뀐 핵심은 세 가지다.

- suggestion phase filtering이 registry 기반 canonical phase mapping을 직접 읽는다
- `prepare_harness_session`와 overlay routing이 ghost alias 없이 visible surface 기준으로 정규화된다
- engine / MCP 내부 대형 모듈이 더 작은 파일들로 분해돼 drift hot-spot과 유지비를 줄인다

## Control-Plane Drift Hardening

이전 상태에서는 tool 존재 여부, phase 분류, prompt literal, bootstrap
entrypoint filtering이 서로 다른 수동 표에 기대는 구간이 있었다.

이번 릴리스에서 고정한 내용:

- unknown tool은 suggestion 후보로 다시 나오지 않음
- prompt에 쓰이는 workflow/tool literal은 registry-visible canonical name만 사용
- bootstrap preferred entrypoint는 unknown/invisible 값을 자동으로 걸러냄
- phase 추론은 follow-up chain을 끊지 않도록 recent-tool trail 기준으로 보정

즉, 이전 이름이나 숨은 alias가 control plane에 새는 drift를 줄이고, 같은 종류의
회귀가 테스트에서 바로 걸리도록 바꿨다.

## Internal Modularization

이번 태그 범위에는 engine/MCP 내부 분해도 같이 포함된다. 목표는 public contract를
바꾸는 것이 아니라, 너무 큰 파일에 몰렸던 책임을 support / catalog / snapshot /
prepare / corpus / scoring 같은 하위 모듈로 나누는 것이다.

대표 범위:

- `codelens-engine`: `file_ops`, `import_graph`, `project`, `symbols`
- `codelens-mcp`: `surface_manifest`, `tool_defs`, `response_support`,
  `report_jobs`, `telemetry`, `project_ops`, `host_adapter`, `lsp`

결과적으로 review, bootstrap, release verification 경로에서 읽기/검증 단위가 더
작아졌다.
