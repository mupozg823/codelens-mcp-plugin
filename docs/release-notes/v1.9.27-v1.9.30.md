# v1.9.27 – v1.9.30 Release Notes

Release-pipeline hardening window. No behavior change in the MCP server or engine runtime; every tag in this window ships the same `codelens-mcp` binary shape as `v1.9.26` with progressively stricter release-artifact plumbing.

## Scope

| Tag        | Commit    | Fix                                                           |
| ---------- | --------- | ------------------------------------------------------------- |
| `v1.9.27`  | `67712de` | `fix(release): package ghcr image from release artifact`      |
| `v1.9.28`  | `7626df5` | `fix(release): skip airgap bundle when model is unstaged`     |
| `v1.9.29`  | `dceaa7f` | `fix(release): relax pem validation for signed artifacts`     |
| `v1.9.30`  | `dcd5958` | `fix(release): accept non-empty cert sidecars`                |

## Why

Each fix closes a release-workflow edge case that only manifested during the actual tagged publish, not during CI dry-runs:

- **GHCR image packaging** — the Linux OCI image is now built from the published release archive instead of a separate CI checkout, guaranteeing the image payload matches the signed tarball.
- **Airgap bundle gating** — when the bundled embedding model is not staged, the airgap archive step is skipped rather than failing the release.
- **PEM sidecar validation** — Fulcio-issued signing certs are accepted with relaxed parsing; the earlier strict match rejected otherwise-valid certs.
- **Non-empty cert sidecars** — `.pem` sidecars are now accepted as long as they contain a certificate body, even when the file metadata differs from the signature path’s expectation.
