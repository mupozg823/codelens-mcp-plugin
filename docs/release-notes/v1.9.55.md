# v1.9.55

## Summary

v1.9.55는 runtime shape를 바꾸는 릴리스가 아니다. `v1.9.54`의 automated
harness / strict coordination / registry reduction은 그대로 유지하고,
public-facing product surface를 다시 **CodeLens-first** 기준으로 정리한
patch release다.

핵심은 세 가지다.

- public primary name을 다시 `CodeLens MCP`로 고정
- parked rebrand / migration 문서를 active roadmap가 아니라 archived draft로 명시
- generated surface manifest와 agent-experience resource가 현재 public naming
  policy를 정확히 반영하도록 정합화

## Public Surface Cleanup

이번 릴리스에서 바뀐 표면은 아래와 같다.

- root `README.md`
- `docs/index.md`
- `docs/platform-setup.md`
- `docs/architecture.md`
- `docs/architecture-d3.html`
- `codelens://design/agent-experience`
- `docs/generated/surface-manifest.json`

이제 active 문서에서는 `CodeLens MCP`가 public primary name으로 일관되게
보이고, future rename proposal은 archived 문서로 분리된다.

## Archived Rebrand Docs

아래 문서들은 active cutover plan이 아니라 **historical design exploration**
로 표시된다.

- `docs/adr/ADR-0007-symbiote-rebrand.md`
- `docs/design/symbiote-ux-flows-v1.md`
- `docs/design/symbiote-phase3-rename-plan.md`
- `docs/migrate-from-codelens.md`

즉, `symbiote://` / `SYMBIOTE_*` runtime compatibility alias는 남아 있어도,
public roadmap와 install guidance는 더 이상 Symbiote cutover를 전제로 쓰지
않는다.

## Agent Experience Resource

`codelens://design/agent-experience`는 이제 active doc path
`docs/design/codelens-agent-flows-v1.md`를 가리킨다.

resource payload의 naming policy도 현재 상태에 맞게 정리됐다.

- `public_primary_name = CodeLens MCP`
- `public_binary_name = codelens-mcp`
- `public_workspace_prefix = codelens`
- `public_name_status = codelens_primary`
- `compatibility_aliases = [symbiote://, SYMBIOTE_*]`

즉, public name과 technical compatibility boundary를 혼동하지 않는 shape로
바뀌었다.

## Verification

이번 릴리스 컷 전에 확인한 검증은 아래와 같다.

```bash
python3 scripts/surface-manifest.py --write
cargo test -p codelens-mcp --features http
```

최종 Rust HTTP suite 결과:

- `571 passed`
- `0 failed`

즉, CodeLens-first public surface 복원 이후에도 host adapter, routing policy,
surface manifest, workflow/resource contract, strict coordination 경로는 회귀되지
않았다.
