# v1.9.21 Release Notes

## Highlights

- **6-language benchmark matrix**: Go (gin) and Java (gson) datasets added, completing Rust/Python/TS-JS/Go/Java/C coverage
- **OpenTelemetry OTLP exporter**: opt-in `otel` feature ships spans to Jaeger, Grafana Tempo, or any OTLP-compatible collector
- **486 tests passing** across engine (286) + mcp (210) + mcp-no-semantic (190)

## New Features

### Go + Java Benchmark Datasets

- `embedding-quality-dataset-gin.json` — 30 queries targeting gin-gonic/gin (Go web framework)
- `embedding-quality-dataset-gson.json` — 30 queries targeting google/gson (Java JSON library)
- Matrix registry updated in `embedding-quality-matrix.py`

### OpenTelemetry Exporter (`--features otel`)

- Feature-gated OTLP gRPC exporter via `tracing-opentelemetry` + `opentelemetry-otlp`
- Activated by setting `CODELENS_OTEL_ENDPOINT` (e.g., `http://localhost:4317`)
- Layered subscriber: stderr fmt + OTel layer coexist
- Graceful fallback: if endpoint is unset or exporter init fails, stderr-only tracing continues
- Zero impact on default binary size — `otel` feature is opt-in

## Build

```bash
cargo build --release                          # standard (no OTel)
cargo build --release --features otel          # with OTel exporter
cargo build --release --features http          # with HTTP transport
cargo build --release --features http,otel     # both
```
