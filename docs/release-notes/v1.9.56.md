# v1.9.56

## Summary

v1.9.56은 기능 릴리스가 아니다. `v1.9.55`의 runtime shape를 그대로 유지하면서,
GitHub와 crates.io에 노출되는 설명 메타데이터를 실제 아키텍처에 맞게 정정한
patch release다.

핵심은 세 가지다.

- `codelens-mcp`는 MCP server로
- `codelens-engine`은 순수 Rust 분석 엔진으로
- `codelens-tui`는 운영/가시화 TUI로

각 crate의 역할이 메타데이터와 README 첫 화면에서 바로 읽히도록 다시 썼다.

## Metadata Corrections

이전 릴리스에서는 workspace description을 세 crate가 그대로 공유했다.
그래서 `codelens-engine`까지도 crates.io에서 `MCP server`처럼 보이는 문제가
있었다.

이번 릴리스에서 정정한 항목:

- `Cargo.toml` workspace description
- `crates/codelens-engine/Cargo.toml` crate-specific description
- `crates/codelens-tui/Cargo.toml` crate-specific description
- `crates/codelens-mcp/README.md` release line
- crate README 상단 copy
- GitHub repository description

정정 후 short description은 아래와 같다.

- `codelens-mcp`:
  `Rust MCP server for bounded code intelligence, gated mutation, and auditable agent workflows`
- `codelens-engine`:
  `Pure Rust code intelligence engine for repository indexing, graph analysis, and local semantic retrieval`
- `codelens-tui`:
  `Terminal dashboard for CodeLens indexes, symbol search, and operator health`

## Why This Matters

이 릴리스는 문구 수정처럼 보여도 실제 배포 품질과 직접 연결된다.

- crates.io 설명이 틀리면 install channel 선택이 왜곡된다
- engine crate가 server처럼 보이면 라이브러리 소비자가 잘못된 기대를 갖는다
- GitHub repo description이 오래된 marketing copy에 묶여 있으면 release / crates /
  docs 표면이 서로 다른 제품처럼 보인다

즉, `global restore`나 action queue가 어떤 채널을 타든 동일한 제품 정체성을
읽게 만드는 정합성 수정이다.
